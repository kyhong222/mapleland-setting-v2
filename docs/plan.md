| 초보자, 전사, 법사 | `DEX×0.8 + LUK×0.5 + 명중률` |

### 회피율
| 직업군 | 공식 |
|---|---|
| 초보자, 전사, 법사, 궁수, 도적, 스트라이커 | `DEX×0.25 + LUK×0.5 + 회피율` |

## 2. 몬스터 UI (vs 성능)

기본정보(HP/EXP/방어/공격/명중/회피 등)는 이미 표기됨. **필요명중률 · 명중확률 · 물리회피확률 · 마법회피확률**을 추가한다.

**출처**: v1 `mapleland-setting/src/components/DamageReceivedTable.tsx`
**대상 파일**: `src/components/panels/MonsterPanel.tsx`(`VsPerformance`), 신규 `src/domain/combat.ts`

### 입력값 (플레이어)
- **캐릭터 명중치 `charAccuracy`**
  - 마법사: `장비 마법명중(macc) + floor(INT/10) + floor(LUK/10)`
  - 그 외: `합산 명중률(effects.acc: 장비+버프+패시브+마스터리) + 스탯보정`
    - 마스터리/패시브 acc는 별도 항으로 더하지 않는다 (이미 effects.acc에 합산됨).
    - 스탯보정은 1번 항목의 직업별 명중 공식(DEX/LUK 부분)을 정본으로 사용.
- **캐릭터 회피치 `totalEva`** = 1번 항목의 직업별 회피 공식 결과(합산 eva + DEX/LUK 보정)

> 명중/회피의 스탯유래분은 **1번 세분화 공식이 정본**. v1의 2분류(warrior/mage vs archer/thief)는 참고만.

### 명중확률 `hitRate` (%)
- `monsterEVA ≤ 0` → **100**
- 마법사: `{마법명중치 / (몬스터회피치 + 1)} × {1 + 0.0415 × (캐릭터레벨 − 몬스터레벨)}`
- 그 외: `{명중치 / ((11/6 + 0.07×D) × 몬스터회피치)} − 1`, `D = max(0, 몬스터레벨 − 캐릭터레벨)`
- 결과 `× 100` 후 `clamp[0, 100]`

### 필요명중률 (100% 명중 역산)
- `monsterEVA ≤ 0` → **0**
- 마법사: `ceil((몬스터회피치 + 1) / modifier)`, `modifier = 1 + 0.0415×(레벨 − 몬레벨)` (≤0 → ∞)
- 그 외: `round(2 × (11/6 + 0.07×D) × 몬스터회피치)`

### 물리 회피확률 (%)
- `rate = totalEva / (4.5 × monsterACC) × 100`
- `monsterACC ≤ 0` → max
- clamp: **min = 도적 5 / 그 외 2**, **max = 도적 95 / 그 외 80**

### 마법 회피확률 (%)
- `rate = (10/9 − monsterACC / (0.9 × totalEva)) × 100`
- `monsterACC ≤ 0` 또는 `totalEva ≤ 0` → max
- clamp: 물리와 동일

### 페이크 추가회피 (채택)
- 물리/마법 회피확률 모두 독립 판정: `최종 = 1 − (1 − base) × (1 − 페이크 evaP)`
- 페이크 evaP는 스킬 레벨/스탯 데이터에서 가져옴.
- (회피 10당 증가량은 보류)

## 3. 공격력 계산

케이스별 여러 표기 지원. **출처**: v1 `mapleland-setting/src/components/DamageTable.tsx`
**대상 파일**: `src/domain/attackPower.ts`, `src/components/panels/AttackPanel.tsx`

### 공통 입력값
- `P` 주스탯, `S` 부스탯, `A` 총공격력, `M` 숙련도비율(=숙련도%/100)
- **부스탯 보정**: 도적은 `부스탯 + (pureStr + equipStr)`
- **숙련도** = `(마스터리1.mastery + 10) + 마스터리2.mastery`
  - 마스터리2: 활=보우엑스퍼트 / 석궁=크로스보우엑스퍼트 / 창·폴암=비홀더
- **무기상수(Cmin/Cmax)**: 한손검4/4, 한손도끼·둔기 3.2/4.4, 두손검4.6/4.6, 두손도끼·둔기 3.4/4.8, 창·폴암 3/5, 활3.4, 석궁3.6, 단검3.6, 아대3.6 (건·너클·완드·스태프는 v1 미정 → 추후)

### 표기 케이스 (비마법사) — **포함 대상만**
| 케이스 | MIN | MAX | 표시 조건 |
|---|---|---|---|
| 스탯창 공격력 | `⌊(P·Cmin·0.9M + S)·A/100⌋` | `⌊(P·Cmax + S)·A/100⌋` | 항상 |
| 최대 공격력(베기) | `⌊(P·Cmax·0.9M + S)·A/100⌋` | `⌊(P·Cmax + S)·A/100⌋` | Cmin≠Cmax |
| 최소 공격력(찌르기) | `⌊(P·Cmin·0.9M + S)·A/100⌋` | `⌊(P·Cmin + S)·A/100⌋` | Cmin≠Cmax |
| 트스/럭세 | `⌊LUK·2.5·A/100⌋` | `⌊LUK·5·A/100⌋` | 아대(claw) |

> **베놈·로어는 제외** (스코프 아웃).

무기별 베기/찌르기 라벨:
- 도끼·둔기류: MAX="베기 공격력" / MIN="찌르기 공격력"
- 창: MAX="찌르기 공격력(스피어 버스터)" / MIN="베기 공격력(드래곤 쓰레셔: 창)"
- 폴암: MAX="베기 공격력(드래곤 쓰레셔: 폴암)" / MIN="찌르기 공격력(폴암 버스터)"
- 그 외: "최대/최소 공격력"

### 마법사
- `magicMax = ⌊(MAD²/1000 + MAD)/30 + INT/200⌋`
- `magicMin = ⌊(MAD²/1000 + MAD·0.9M)/30 + INT/200⌋`
- 총 마력(MAD) 별도 표시

### 공탯비 (포함) — 각 케이스별 1공↔1스탯 교환비 (평균 기대값)
- 일반: `K` 기준 → `atkToStat = (P·K + 2S)/(K·A)`, `statToAtk = 역수`
  - 스탯창 K=`Cmin·0.9M + Cmax` / 최대공 K=`Cmax·(0.9M+1)` / 최소공 K=`Cmin·(0.9M+1)`
- 트스/럭세: `atkToStat = P/A`, `statToAtk = A/P`
- 마법사: `α=(1+0.9M)/2`, `1마력 마진 = (2MAD/1000 + α)/30`, `1인트 마진 = 1마력마진 + 1/200`

### 몬스터 방어 반영 실질 공격력 (신규)
스탯창 공격력 결과에 몬스터 방어·렙차를 적용해 실질 최소~최대 표시.
- `D = max(0, 몬스터레벨 − 캐릭터레벨)` (캐릭이 높으면 0)
- 결과 음수면 **1로 고정**

**물리** (WDEF = 몬스터 물리방어력)
- `MAX = max(1, ⌊최대스공 × (1 − 0.01·D) − WDEF × 0.5⌋)`
- `MIN = max(1, ⌊최소스공 × (1 − 0.01·D) − WDEF × 0.6⌋)`

**마법** (MDEF = 몬스터 마법방어력)
- `MAX = max(1, ⌊최대스킬뎀 − MDEF × 0.5 × (1 + 0.01·D)⌋)`
- `MIN = max(1, ⌊최소스킬뎀 − MDEF × 0.6 × (1 + 0.01·D)⌋)`

> 방향 주의: v1 `DamageReceivedTable`의 방어식은 몬스터→캐릭터(받는 데미지)용. 위 식은 캐릭터→몬스터(주는 데미지)용.

  - 소울마스터→전사식 / 플레임위자드→법사식 / 윈드브레이커→궁수식 / 나이트워커→도적식.
  - **스트라이커**: 명중 `DEX×0.6+LUK×0.3`, 회피 `DEX×0.25+LUK×0.5` (인파이터 유사하나 해적 1차 기준).
- **장비 제한**: 대응 클래스와 **동일 비트마스크** (소울마스터=전사 장비 등).
- **최대 레벨 120**, 10~70 구간 레벨업 시 추가 AP +1.

### 총 AP 공식 (검증 완료)
```
총AP(L) = 19 + 5×L + max(0, min(L,70) − 9) + (L≥70 ? 5 : 0) + (L≥120 ? 5 : 0)
```
- `5×L` 레벨당 5 / `max(0, min(L,70)−9)` 10~70 +1(최대 61) / `≥70:+5`,`≥120:+5` 전직 보너스 / `19` 상수
- 검증: L62 → 382 (=298+76+4+4), L120 → 690 (=602+80+4+4) 일치.

### 스킬/패시브
- 전체 스킬 정보는 **추후 추가**. 이번 스코프는 직업·스탯·AP·명중/회피·마스터리까지.

## 6. 피격데미지 UI (신규, v1 이관)

**출처**: v1 `mapleland-setting/src/components/DamageReceivedTable.tsx`
v2에서 **새 UI 섹션**으로 표기.

### 물리 접촉/물리스킬 피격 데미지
- `C(전사) = STR/2800 + DEX/3200 + INT/7200 + LUK/3200`
- `C(그 외) = STR/2000 + DEX/2800 + INT/7200 + LUK/3200`
- `A = C + 0.28`
- `t = 몬스터ATT² × rand(0.008 ~ 0.0085)`
- `D(캐릭Lv ≥ 몬Lv) = 13 / (13 + 캐릭Lv − 몬Lv)`, `D(캐릭Lv < 몬Lv) = 1.3`
- `B(PDD ≥ stdPDD) = C×28/45 + 캐릭Lv×7/13000 + 0.196`
- `B(PDD < stdPDD) = D × (C + 캐릭Lv/550 + 0.28)`
- **데미지 = `max(1, ⌊t − PDD×A − (PDD − stdPDD)×B⌋)`**
- `PDD` = 캐릭터 물방(장비 pdef + 방패마스터리 + 방어버프), `stdPDD` = 직업·레벨별 기준방어력(standardPDD.json)

### 마법 피격 데미지
- `t = 몬스터MATT² × rand(0.0075 ~ 0.008)`
- `defense = (MDD/4 + STR/28 + DEX/24 + LUK/20) × K`, `K = 1.2(마법사) / 1.0(그 외)`
- **데미지 = `max(1, ⌊t − defense⌋)`**
- `MDD` = 캐릭터 마방(장비 mdef + totalINT + 방어버프)

### 공통 수정자 (applyModifiers)
1. 스킬 데미지 감소(속성저항·파워가드 등) multiplier
2. 파워업/매직업: 일반 ×1.15 / 보스 ×1.3
3. 메소가드: 50% 흡수 (파워업 적용 **전** 기준)
4. 최종 = `max(1, ⌊파워업후 − 메소흡수⌋)`

### 표시 범위 & 필요 데이터
- 물리 접촉 + 몬스터 스킬별(WZ 데이터) 속성별 피격 데미지(무/불/얼음/번개/독).
- 필요: 몬스터 ATT·MATT, mob WZ 스킬 데이터, standardPDD 테이블, 방어버프/특수스킬(메소가드·저항).
- **확인 필요**: 위 데이터가 v2에 있는지 / v1에서 이관해야 하는지.

## 7. 방컷 UI (신규)

특정 공격스킬 기준으로, 선택된 몬스터를 **1~5방 안에 처치할 확률**을 계산.

### 입력 (의존)
- 1방당 실질 데미지 범위 = **3번(몬스터 방어 반영 실질 공격력)** 결과.
- 스킬 정보(타수·데미지 계수) = 공격 스킬 DB.
- 몬스터 HP (`src/domain/monster.ts` `maxHP`).

### 개념
- 1방 데미지 분포로 N방 누적 데미지 ≥ 몬스터 HP 확률 = "N방컷 확률". 1~5방 각각 표시.

### 상태
- **데미지 계산 방식 / 확률 공식: 추후 제공 (보류)** — 확정 전까지 착수 대기.
- 의존: 3번 + 공격 스킬 DB 선행.

## 데미지 계산 파이프라인 (확정, ayumilove 10단계 기준)

전체 데미지는 아래 10단계 순서로 계산한다. 스킬 데이터(`src/data/skills`)의 필드가 각 단계 입력이 된다.

| 단계 | 처리 | 데이터/비고 |
|---|---|---|
| 1 | 기본 공식으로 MIN~MAX 산출 | 물리=무기공식(`calcPhysical`) / 마법=`calcMagic`(스킬 `mad`=Spell Attack) / 특수식 |
| 2 | 속성 배율 곱 | 스킬 `elementalAttribute` vs 몹 속성 → **약점 1.5× / 반감 0.5× / 무효 0×** |
| 3 | 몬스터 방어 감소 | 물리 `스공×(1−0.01D)−WDEF×0.5(min은 0.6)`, 마법 `−MDEF×0.5×(1+0.01D)`. 소환수/어썰터(동급↑)는 **방무 스킵** |
| 4 | range에서 랜덤 X 추출 (비정수) | |
| 5 | 스킬 배율 × | **물리 = 스킬 `damage%`** (기본공격 100%), **마법 = 100%**(계수는 1단계 `mad`에 반영) |
| 6 | 크리티컬 보너스 | **크리 배율 = 스킬% + 크리스킬% + 샤프아이즈(140) − 100** (−100은 1회, 없는 항 제외) |
| 7 | X × (5·6단계 배율) | |
| 8 | `[1, 199999]` clamp | ※ 메이플랜드 상한 199999 |
| 9 | After-modifier × | 타격별 배율 다른 스킬(예: **바이퍼 피스트**) — **미구현(=1)**, 필요 시 스킬별 예외 |
| 10 | 정수부(floor) | |

### 크리티컬 공식 검증
- 럭세(150) + 크리티컬스로우(200) − 100 = **250%**
- 트스(150) + 크리티컬스로우(200) + 샤프(140) − 100 = **390%**
- 샤프아이즈 실적용 = **+140%**(툴팁 40% 아님), 크리는 크리확률만큼만 발동(평균은 확률 가중)

### 특수 스킬 공식 (step 1 예외) — 필요 시 개별 구현
- **드래곤 로어**(다크나이트): `(STR×4.0+DEX)×WA/100`, MIN은 STR항에 `×숙련도×0.9`
- **힐 데미지**(비숍): `(INT×1.2+LUK)×Magic/1000×타겟배수`, 타겟배수 `1.5+5/N`, 스킬%는 툴팁값(최대 300%)
- **파워 너크백**(캡틴): `(DEX×3.4…)×WA/150`, 마스터리 0.1 고정
- **소환수**(피닉스/프로스트프레이/옥토퍼스/가비오타): `(DEX×2.5+STR)×AtkRate/100`, **방무**
- **베어핸드/클로펀치/블랭크샷** 등 맨손·특수: 개별식
- **DoT**(포이즌미스트/파벤/닌자앰부시/섀도우웹): 초당 틱 `몹HP/(N−Lv)` 등
- **에로우봄**: 직격 50% / 스플래시 130% (대상별 스킬% 상이)

### 물리/마법 분기 요약
- **물리 스킬** = 표준 무기공식(1단계) × `damage%`(5단계)
- **마법 스킬** = Spell 공식에 `mad` 투입(1단계), 5단계 배율 100%
